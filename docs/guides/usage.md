## 2. 环境准备与启动系统

### 2.1 推荐环境

- 推荐 Python：`3.11`
- 推荐先准备好 Conda 环境，例如 `study_agent`
- 首次运行前确保依赖已安装，并准备好 `.env`

最小 `.env` 说明请见 `README.md`，完整参数请见 `docs/guides/config-overview.md`。

### 2.2 启动顺序

每次使用前需要同时启动后端和前端两个进程。

```bash
# 终端 1：后端（FastAPI）
conda activate study_agent
python -m backend.api

# 终端 2：前端（Streamlit）
conda activate study_agent
streamlit run frontend/streamlit_app.py
```

看到这些标志信息说明启动成功：

- 后端：`Uvicorn running on http://0.0.0.0:8000`
- 前端：浏览器自动打开 `http://localhost:8501`

### 2.3 首次启动时会发生什么

后端启动时会做几件事：

1. 恢复已有课程工作区
2. 如果 `EMBEDDING_PRELOAD_ON_STARTUP=1`，预热嵌入模型
3. 如果启用了 rerank，并且配置允许预热，则预热 reranker

这意味着当前版本的嵌入模型冷启动已经尽量前移到服务启动阶段，而不是等第一个检索请求才加载。  
如果预热失败，服务仍然会继续启动；只是第一次真正触发检索时，可能会再经历一次模型加载等待。

如果你需要后台维护能力，可以单独启动 worker：

```bash
# 可选：会话清理 worker
conda activate study_agent
python -m scripts.workers.session_cleanup_worker

# 可选：在线 shadow eval worker
conda activate study_agent
python -m scripts.workers.shadow_eval_worker
```

默认使用场景下，只启动后端和前端也可以正常对话、检索和生成练习/考试内容；worker 更偏向后台维护和持续评测。

### 2.4 常见启动提示

- 前端必须在后端之后启动，否则容易看到 `Connection refused`
- 首次下载嵌入模型或首次加载 GPU 模型时，启动会更慢
- 如果你使用的是 CPU，索引构建和检索会明显慢于 GPU

## 3. 课程、文件与会话管理

### 3.1 创建课程

1. 打开 `http://localhost:8501`
2. 在侧边栏点击“创建新课程”
3. 填写课程名称和学科标签
4. 点击创建

课程创建成功后，系统会在 `data/workspaces/<课程名称>/` 下初始化目录。

当前主要目录包括：

```text
uploads/      原始教材
index/        向量索引
notes/        保存的笔记
mistakes/     错题记录
practices/    练习记录
exams/        考试记录
sessions/     服务端会话状态
```

### 3.2 切换课程

侧边栏下拉菜单选择已有课程即可切换。  
切换课程后：

- 文件列表切换
- 索引状态切换
- 对话上下文切换
- 会话状态切换

### 3.3 文件管理

侧边栏“文件与索引”面板支持：

| 操作 | 说明 |
|---|---|
| 查看文件列表 | 显示文件名、大小、修改时间 |
| 删除单文件 | 仅删除该原始文件 |
| 构建 / 重建索引 | 基于当前所有上传文件重建索引 |
| 删除索引 | 仅删除索引，不删除原始文件 |

### 3.4 会话管理

当前前端已经支持两种会话管理操作：

| 操作 | 说明 |
|---|---|
| `清理过期会话` | 调用后端接口，删除超过 TTL 的 `sessions/*.json` |
| `删除当前会话` | 删除当前正在使用的会话状态，并清空当前聊天历史 |

这两项都不会删除课程本身，也不会删除 `practices/`、`exams/`、`notes/` 等用户记录。

### 3.5 删除课程的当前状态

当前版本已经支持：

- 删除单个文件
- 删除索引
- 删除当前会话
- 清理过期会话

但**还没有对外提供完整的“删除整个课程工作区”正式入口**。如果你确实需要删除课程，建议先完整备份 `data/workspaces/<课程名称>/`，再在明确知道后果的前提下做人工清理。

## 4. 上传教材、构建索引与引用来源

### 4.1 支持的文件格式

| 格式 | 说明 |
|---|---|
| `.pdf` | 文字版 PDF |
| `.txt` | 自动识别常见编码 |
| `.md` | Markdown 文档 |
| `.docx` | Word 文档 |
| `.pptx` | PowerPoint 文档 |
| `.ppt` | 旧版 PowerPoint，先转换后解析 |

### 4.2 上传步骤

1. 选择课程
2. 在侧边栏上传文件
3. 等待上传完成
4. 上传完成后点击“构建索引”

### 4.3 为什么上传后还要构建索引

上传文件只是把原始教材放进 `uploads/`。  
只有构建索引之后，系统才会：

1. 解析文本
2. 切块
3. 生成向量
4. 写入 `index/faiss_index.faiss` 和 `index/faiss_index.pkl`

也就是说：**上传完成不等于可检索，必须构建索引。**

### 4.4 首次构建时会发生什么

- 可能需要下载嵌入模型 `BAAI/bge-base-zh-v1.5`
- 如果机器有可用 GPU，会自动使用 CUDA
- CPU 环境下构建时间会更长

### 4.5 什么时候必须重建索引

以下场景建议重建：

1. 新增教材文件
2. 替换教材文件
3. 删除教材文件后，希望检索结果同步更新
4. 你怀疑当前检索结果和教材不一致

### 4.6 批量重建索引

当前批量重建脚本路径已经迁到：

```bash
py -3.11 scripts/rebuild_indexes.py
```

### 4.7 教材准备建议

推荐：

- 用有意义的文件名，例如 `线性代数_第三章_线性方程组.pdf`
- 优先使用可复制文字的 PDF
- PPT 最好整理好标题和每页文本

不推荐：

- `文档1.pdf`
- `新建 Microsoft Word 文档.docx`
- 纯扫描 PDF

### 4.8 引用来源是怎么来的

当前回答中的引用来自 RAG 检索结果，通常会展示：

- 文档名
- 页码
- 引文片段
- 向量相似度
- BM25 分数
- 融合排序分

要注意三点：

1. `融合排序分` 是 RRF 排序分，不是“语义相关度”
2. `BM25 分数` 是词法匹配分，数值常常会比向量分大很多，这是正常现象
3. 现在系统有证据门槛，如果候选片段太弱，会直接判定“检索未命中”

### 4.9 “教材未涉及”是什么意思

当前系统有两类相关提示：

- “教材未涉及”
- “未找到可靠教材片段”

这通常表示：

1. 当前问题确实不在课程教材范围内
2. 检索结果太弱，没有通过证据门槛
3. 当前课程选错了，或者索引尚未建立

这类提示不是坏事，它意味着系统没有把弱证据硬冒充成教材依据。

## 5. 学习模式

### 5.1 适用场景

学习模式适合：

- 理解新概念
- 复习知识点
- 查找定理、公式、机制
- 梳理知识脉络
- 生成总结与思维导图

### 5.2 一次完整流程

```text
用户提问
  -> Router 规划
  -> 教材检索
  -> 按需记忆预取
  -> Tutor 讲解
  -> 如有需要调用工具
  -> 返回带引用的回答
```

### 5.3 提问示例

```text
什么是矩阵的秩？
请结合教材第三章，解释线性方程组何时有唯一解
介绍一下多头自注意力机制
帮我整理从向量空间到矩阵秩的知识脉络
帮我生成第二章的思维导图
```

### 5.4 你会看到什么输出

学习模式的典型回答包括：

- 核心答案
- 分步骤解释
- 易错点或延伸说明
- 教材引用来源
- 必要时的工具结果

### 5.5 学习模式与教材引用

- 如果教材命中且证据通过门槛，会看到引用面板
- 如果教材未命中，会明确说明“未找到可靠教材片段”
- 如果问题超出教材范围，但工具能补足，系统也会说明回答依据来自其他途径

### 5.6 与历史记录和薄弱点的关系

学习模式下：

- 用户画像会默认作为轻量背景注入
- 情景记忆只在有明确回顾 / 历史 / 弱项等意图时按需检索

你可以主动这样提：

```text
回顾我之前关于线性相关的错题
结合我过去的薄弱点，帮我总结这一章
根据我之前练习里最容易错的知识点，再讲一遍 Layer Norm
```

### 5.7 常见误用

不太推荐：

- 问得过于宽泛，例如“讲讲整本书”
- 不指定章节、术语或目标教材
- 把明确的出题/评分任务放在学习模式里

## 6. 练习模式

### 6.1 适用场景

练习模式适合：

- 单题巩固
- 多题训练
- 针对薄弱点做强化练习
- 做题后立即获得讲评

### 6.2 练习模式的两种常见形态

1. 单题练习
2. 多题练习

当你明确要求多题时，系统会走试卷型 artifact 链路，但仍然归类在练习模式下。

### 6.3 一次完整流程

```text
第一步：让 AI 出题
第二步：AI 返回题目
第三步：你提交答案
第四步：AI 评分并讲评
第五步：结果自动保存到 practices/ 和记忆库
```

### 6.4 提问示例

```text
给我出一道关于矩阵秩的中等难度题
来一道困难的综合证明题
针对第三章线性方程组出三道题
根据我之前的薄弱点，再出一组 Attention 练习
```

### 6.5 提交答案示例

```text
(1) rank(A)=2
(2) 线性相关，因为第二行是第一行的2倍
```

或者多题答案：

```text
1.A
2.正确
3.B
4.简答：我认为 Layer Norm 的作用是...
```

### 6.6 练习模式下你会看到什么

- 题目正文
- 评分结果
- 逐题讲评
- 得分
- 错误类型 / 薄弱点提示

评分完成后，系统会自动：

- 保存 Markdown 记录到 `practices/`
- 将评分事件写入 `memory.db`
- 更新用户画像中的薄弱点

### 6.7 练习模式的引用语义

练习题目和讲评都可能使用教材引用，但不是每次都会出现引用面板。  
如果系统明确说没有找到可靠教材片段，说明当前题目或讲评不是严格建立在可靠教材证据上。

### 6.8 常见误用

- 只让 AI 出题，但不提交答案，就期待 `practices/` 自动有评分记录
- 在单题练习里提交模糊答案，导致系统难以判断是否进入评分分支
- 一次性混合很多题型要求，又不给范围

## 7. 考试模式

### 7.1 适用场景

考试模式适合：

- 期末前自测
- 模拟整套试卷
- 一次性完成后再统一批改

### 7.2 与练习模式的区别

| 对比项 | 练习模式 | 考试模式 |
|---|---|---|
| 题目形态 | 单题或多题训练 | 综合试卷 |
| 用户节奏 | 出题后即可立即评分 | 更强调“先完成，再统一提交” |
| 记录位置 | `practices/` | `exams/` |
| 检索范围 | 默认较窄 | 默认更宽，top-k 更大 |

### 7.3 一次完整流程

```text
第一步：请求系统出卷
第二步：系统返回试卷
第三步：你完成后一次性提交答案
第四步：系统批改并生成考试报告
第五步：结果保存到 exams/ 和记忆库
```

### 7.4 使用示例

```text
来一套线性代数第一章综合测试
给我一套关于 Transformer 基础的模拟考试
出一套 90 分钟难度的复习卷
```

### 7.5 提交答案建议

考试模式更适合一次性提交完整答案，而不是边做边交。  
如果题量较多，建议保持清晰编号。

### 7.6 考试报告中通常包含什么

- 总分或整体评价
- 分题讲评
- 薄弱知识点
- 改进建议

### 7.7 记录与后续复习

考试完成后，系统会：

- 保存考试记录到 `exams/`
- 把错误与薄弱点写入长期记忆
- 在后续学习 / 练习中更可能围绕这些薄弱点展开

## 8. 工具与用户可见行为

工具由系统自动调用，用户通常不需要手动指定。理解它们的作用，有助于判断“为什么这轮回答会这样”。

### 8.1 计算器

适用场景：

- 数值计算
- 评分链路中的分数汇总
- 含明确数学运算的问题

示例：

```text
计算 det([[2,1],[5,3]])
```

### 8.2 网页搜索

适用场景：

- 问题超出教材范围
- 需要时效性信息
- 需要教材外背景补充

注意：

- 需要配置 `SERPAPI_API_KEY`
- 失败时系统应显式说明“联网搜索失败”或“本次未能获取外部资料”

### 8.3 文件写入

适用场景：

- “保存到笔记”
- “整理成一份 Markdown 笔记”
- “把这段总结写下来”

输出位置：

`data/workspaces/<课程名>/notes/`

### 8.4 思维导图

适用场景：

- 章节总结
- 知识脉络梳理
- 复习提纲

前端通常支持导出：

- SVG
- PNG
- `.mmd`

### 8.5 时间查询

适用场景：

- 询问今天的日期、时间、星期
- 时间相关上下文需求

### 8.6 memory_search

这一项更接近后台个性化能力，而不是“每轮都应该出现的显式功能”。

它主要用于：

- 回顾过去的错题
- 结合你之前的薄弱点生成新练习
- 在讲解时提醒你历史上容易错的点

你可以主动这样触发：

```text
回顾我关于线性方程组的历史错题
总结我之前所有关于 Layer Norm 的易错点
```

### 8.7 shadow_eval 开关

前端当前有会话级 `shadow_eval` 开关。它的含义是：

- 主回答照常生成
- 系统把当前请求异步送入后台评测队列
- 不阻塞主对话

这更像“在线旁路评测”，不是额外让模型再回答一遍给你看。

## 9. 记录、记忆与数据管理

### 9.1 用户可读记录

主要路径：

```text
data/workspaces/<课程名>/practices/
data/workspaces/<课程名>/exams/
data/workspaces/<课程名>/notes/
data/workspaces/<课程名>/mistakes/mistakes.jsonl
```

### 9.2 长期记忆库

SQLite 数据库路径：

```text
data/memory/memory.db
```

你可以这样理解它：

- `practices/`、`exams/`、`notes/` 主要是给人看的
- `memory.db` 主要是给系统做长期检索和画像更新用的

### 9.3 用户画像与情景记忆的区别

对使用者来说，最简单的理解方式是：

- 用户画像：系统对你学习状态的长期印象，例如“这门课你总在哪些知识点失分”
- 情景记忆：某一次具体练习、错题、考试记录

### 9.4 如何利用历史记录复习

推荐做法：

1. 先看 `practices/` 或 `exams/` 的 Markdown 记录
2. 再让系统总结你最近的薄弱点
3. 然后针对这些点继续学习或练习

示例：

```text
根据我最近三次练习，帮我总结最薄弱的三个知识点
回顾我这门课最近一次考试里的主要错误
```

### 9.5 备份建议

最有价值的备份对象通常是：

- `data/workspaces/<课程名>/`
- `data/memory/memory.db`

## 10. 前端状态提示与引用显示

### 10.1 流式状态提示

当前前端会根据 SSE 事件显示阶段状态，例如：

- 检索中
- 调用工具中
- 整理答案中
- 后端仍在处理中

这意味着“界面暂时没出正文”不一定是卡死，也可能是在处理检索、工具或最终整理。

### 10.2 上下文预算角标

当前前端会显示上下文预算相关提示。它的作用是帮助你理解：

- 当前轮用了多少上下文
- 是否接近预算上限
- 是否发生了较强压缩

如果你发现预算压力很高，通常建议：

1. 缩小问题范围
2. 指定章节
3. 拆分成两轮提问

### 10.3 为什么有时没有引用面板

可能原因包括：

1. 当前轮没有命中可靠教材片段
2. 当前回答主要依赖工具结果
3. 当前课程未构建索引
4. 当前问题和课程教材关系很弱

### 10.4 引用分数字段怎么理解

当前前端展示：

- `重排分`
- `向量相似度`
- `BM25 分数`
- `融合排序分`

建议这样理解：

- 看“重排分”：Cross-Encoder 对候选片段做二阶段精排后的相关性分数
- 看“向量相似度”：语义接近程度的一个线索
- 看“BM25 分数”：词法命中强度
- 看“融合排序分”：RRF 初排阶段的排序依据

不要直接把这四者当成统一量纲，也不要把 `融合排序分` 或 `重排分` 当成“真实相关度百分比”。

### 10.5 为什么系统会明确说“未找到可靠教材片段”

这是当前版本的一项刻意改进。  
以前只要检索器返回了东西，系统就可能把它当引用；现在会经过证据门槛，所以弱证据会被拦掉。

这个提示出现时，通常意味着：

- 系统选择诚实地告诉你“这轮没有可靠教材依据”
- 而不是继续伪造或过度包装引用

## 11. 常见问题与系统化排障

### Q1：上传文件后 AI 说“未找到相关教材内容”

先检查这四件事：

1. 是否已经构建索引
2. 当前课程是否选对
3. PDF 是否是可复制文本的文字版
4. 问题是否确实和当前课程教材有关

### Q2：索引状态异常，明明有文件却查不到

建议按顺序排查：

1. 删除当前索引
2. 重新构建索引
3. 再问一个明确带教材术语的问题

### Q3：扫描 PDF 为什么效果差

因为当前解析主要依赖可提取文本。  
纯扫描 PDF 没有 OCR，就很难被稳定检索。

### Q4：流式回答中途停止

常见原因：

- SSE 中断
- 后端异常
- 外部模型或工具响应异常

先做这两步：

1. 刷新前端页面
2. 查看后端终端日志

### Q5：网页搜索没有触发

可能原因：

1. 没配置 `SERPAPI_API_KEY`
2. 问题本身不需要外部时效信息
3. 当前模式 / 工具策略下没有必要调用

如果你希望更容易触发，可以明确写：

```text
请结合最新资料说明
请联网补充近几年的应用
```

### Q6：工具调用失败了会怎样

当前系统的目标是“显式说明失败”，而不是静默吞掉。  
例如：

- `websearch` 失败，应该说明联网搜索失败
- `memory_search` 失败，不等于“没有记忆”，而是工具本身出了问题

### Q7：练习或考试记录为什么没保存

最常见原因是你还停留在“出题阶段”，并没有真正完成评分。  
一般只有在系统完成评分 / 批改后，才会把记录落盘。

### Q8：为什么这轮回答没有引用面板

这不一定是 bug。  
先判断是不是下面几种情况：

1. 教材未命中
2. 问题在教材外
3. 当前轮主要依赖工具
4. 当前课程索引未构建

### Q9：为什么引用分数看起来很低

尤其是 `融合排序分 0.02 / 0.03` 这种情况，通常是正常的。  
因为它是 RRF 融合排序分，本来量级就比较小。

如果你看到 `重排分`，那是 rerank 阶段的分数，和 `融合排序分` 不是一个量纲，不能直接比较大小。

### Q10：shadow_eval 会不会影响我主对话的速度

当前设计是异步旁路，不应该阻塞主对话。  
如果你担心评测影响使用，可以把前端的 `shadow_eval` 开关关掉。

### Q11：memory 结果为什么有时完全没有

这通常并不表示系统坏了。  
更常见的解释是：

- 当前问题不适合触发情景记忆检索
- 历史里确实没有相关事件
- 当前轮只注入了用户画像，没有注入具体历史片段

## 12. 最佳实践

### 12.1 提问时尽量带术语和范围

更好的提法：

```text
根据第三章解释秩-零化度定理
介绍 Layer Norm，并结合教材说明它和 Batch Norm 的区别
针对注意力机制出三道中等难度练习
```

### 12.2 一轮只做一件核心事

比起在同一轮里同时要求“讲解 + 出题 + 批改 + 生成导图”，更推荐拆成多轮：

1. 先学
2. 再练
3. 再总结

### 12.3 让检索更稳定的方法

1. 指定课程名对应的真实教材术语
2. 指定章节范围
3. 避免太泛的问题

### 12.4 学习闭环建议

一个常见的高效节奏是：

1. 学习模式理解概念
2. 练习模式做针对题
3. 考试模式做综合测试
4. 回到学习模式复盘错题和薄弱点

### 12.5 如何更稳定地生成题目或思维导图

- 生成题目时尽量说明知识点、难度、题量、题型
- 生成导图时尽量说明章节或主题边界

例如：

```text
给我出三道关于线性方程组的中等难度选择题
帮我生成“Transformer 基础结构”思维导图，只覆盖编码器与注意力机制
```
